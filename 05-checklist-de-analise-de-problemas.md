# Checklist de Análise de Problemas

Ao analisar um problema relatado por usuários, é importante seguir um processo estruturado para entender o que está acontecendo.

Este checklist ajuda equipes de suporte a organizar a análise antes de encaminhar ou escalar um problema.

---

## 1. O problema está claro?

Antes de qualquer investigação, confirme se o problema foi descrito de forma compreensível.

Perguntas úteis:

- O que exatamente aconteceu?
- O usuário explicou o que estava tentando fazer?
- Existe contexto suficiente para entender a situação?

Se a descrição estiver vaga, pode ser necessário pedir mais informações.

---

## 2. O problema é reproduzível?

Sempre que possível, tente reproduzir o problema.

Perguntas úteis:

- É possível repetir os mesmos passos?
- O problema ocorre sempre ou apenas em alguns casos?
- Existe um padrão claro de quando ele acontece?

Problemas reproduzíveis são muito mais fáceis de investigar.

---

## 3. Onde o problema ocorre?

Identificar o local exato ajuda a reduzir o escopo da investigação.

Exemplos:

- módulo do sistema
- tela específica
- fluxo de operação
- funcionalidade específica

---

## 4. Qual é o impacto do problema?

Nem todos os problemas têm o mesmo impacto.

Perguntas úteis:

- o usuário consegue continuar o trabalho?
- o problema impede uma ação importante?
- existe uma alternativa ou solução temporária?

Isso ajuda a definir prioridade.

---

## 5. Outros usuários enfrentam o mesmo problema?

Verifique se existem ocorrências semelhantes.

Isso pode indicar:

- um problema isolado
- um erro técnico recorrente
- uma dificuldade de usabilidade
- um padrão de comportamento dos usuários

---

## 6. Existe alguma evidência adicional?

Sempre que possível, reúna informações que ajudem na investigação.

Exemplos:

- prints de tela
- mensagens de erro
- horário da ocorrência
- logs do sistema
- dados utilizados na operação

Essas evidências ajudam equipes técnicas a investigar com mais precisão.

---

## 7. O problema pode indicar uma melhoria?

Nem todo problema é apenas um erro.

Perguntas úteis:

- o fluxo poderia ser mais claro?
- a interface poderia orientar melhor o usuário?
- existe uma funcionalidade ausente?

Muitas melhorias de produto começam a partir de observações feitas pelo suporte.

---

## Conclusão

Seguir um checklist simples ajuda a garantir que problemas sejam analisados de forma consistente.

Isso melhora a comunicação com equipes técnicas, reduz retrabalho e aumenta a qualidade das informações registradas.
