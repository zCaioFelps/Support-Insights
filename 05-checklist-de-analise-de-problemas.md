# Checklist de Análise de Problemas

Este checklist pode ser usado para validar se um problema foi analisado e documentado corretamente antes de ser escalado ou registrado formalmente.

---

## Informações básicas

- [ ] O problema foi descrito de forma clara?
- [ ] Existe contexto suficiente para entender o cenário?
- [ ] O módulo ou funcionalidade afetada foi identificado?

---

## Reprodução do problema

- [ ] Os passos para reproduzir foram descritos?
- [ ] O problema foi reproduzido com sucesso?
- [ ] O comportamento observado foi registrado?

---

## Impacto

- [ ] O impacto para o usuário foi identificado?
- [ ] O problema impede o fluxo de trabalho?
- [ ] Existe alguma solução temporária?

---

## Evidências

- [ ] Prints de tela foram coletados?
- [ ] Mensagens de erro foram registradas?
- [ ] Informações relevantes (horário, dados utilizados) foram anotadas?

---

## Análise inicial

- [ ] Existem outros casos semelhantes?
- [ ] O problema pode indicar dificuldade de usabilidade?
- [ ] Pode ser uma oportunidade de melhoria no produto?

---

## Antes de escalar o problema

Confirme que:

- o problema está bem documentado
- há contexto suficiente para investigação
- informações relevantes foram incluídas

Uma boa análise inicial reduz retrabalho e acelera a investigação técnica.
